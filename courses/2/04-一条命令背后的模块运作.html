用键盘输入一条命令

shell 程序读取你的命令

控制台初始化 tty_init 中，将键盘中断绑定在了 keyboard_interrupt 这个中断处理函数上，也就是说当我们按下键盘，CPU 的中断机制将会被触发，最终执行到这个 keyboard_interrupt 函数中。

先通过 IO 端口操作，从键盘中读取了刚刚产生的键盘扫描码

调用 do_tty_interrupt 函数，将 keyboard_interrupt 时传入的参数 0，作为 tty_table 的索引

  • 0是控制台
  • 1是串行终端1
  • 2是串行终端2

tty_struct结构

struct tty_struct {
    struct termios termios; // 定义了终端的各种模式，包括读模式、写模式、控制模式等
    int pgrp;
    int stopped;
    void (*write)(struct tty_struct * tty);
    struct tty_queue read_q;
    struct tty_queue write_q;
    struct tty_queue secondary;
};

struct tty_queue {
    unsigned long data;
    unsigned long head;
    unsigned long tail;
    struct task_struct * proc_list;
    char buf[TTY_BUF_SIZE];
};

调用copy_to_cooked，在 copy_to_cooked 函数里就是个大循环，只要读队列 read_q 不为空，且辅助队列 secondary 没有满，就不断从 read_q 中取出字符，经过一大坨的处理，写入 secondary 队列里。

对于普通的字符串abcd 这种，对应的处理函数是 do_self

上层进程调用终端的读函数tty_read，将字符从secondary 队列里取走。

进程执行完之后，最终调用到 tty_write 函数将结果输出，会将字符 放入 write_q 这个队列，然后调用 tty 里设定的 write 函数（也就是con_write, console的写函数）。最终会配合显卡，在我们的屏幕上输出我们给出的字符。

进程的阻塞与唤醒

一个进程阻塞的方法非常简单，只需要将其 state 字段，变成非 TASK_RUNNING 也就是非运行态，即可让它暂时不被 CPU 调度。唤醒将对应进程的 state 字段变成 TASK_RUNNING 即可。

  • sleep_on 函数将 state 变为 TASK_UNINTERRUPTIBLE。
  • wake_up 函数将 state 变回为 TASK_RUNNING，也就是 0。

通过每一个当前任务所在的代码块中的 tmp 变量，总能找到上一个正在同样等待一个资源的进程，因此也就形成了一个链表。

每一个进程唤醒后，后半段就是睡眠的状态，这个时候就会将上一个进程唤醒，保证有任务在运行task_running状态。

读硬盘数据全流程

通过一个文件描述符fd，寻找存储在磁盘中的一个文件（具体到磁盘中的哪几个扇区）

sys_read函数，就是把这个 fd 指向的硬盘中的文件，复制到内存中的哪个位置，复制多大。

int sys_read(unsigned int fd,char * buf,int count) {
    ...
}

Linux 下一切皆文件，所以这个函数将管道文件、字符设备文件、块设备文件、目录文件、普通文件分别指向了不同的具体实现。

读取流程

  • 根据文件描述符 fd，在进程表里拿到了 file 信息，进而拿到了 inode 信息。
  • 对 buf 区域的内存做校验
  • 调用具体的 file_read 函数进行读操作

读取硬盘数据的细节

  • getblk 申请缓冲块
  • ll_rw_block 方法负责把硬盘中指定数据块中的数据

当设备的当前请求项为空, 会立即执行该设备的 request_fn 方法, 这个方法会在设备初始化的时候设置。hd_out 就是读硬盘的最底层的函数了，向一堆外设端口做读写操作

在硬盘初始化的时候，将 hd_interrupt 设置为了硬盘中断处理函数，中断号是 0x2E，当发出中断信号就可以开始处理磁盘数据读取
