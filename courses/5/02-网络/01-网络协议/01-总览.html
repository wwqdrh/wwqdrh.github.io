简介

传输层: 流量控制, 在源主机的应用进程与目的主机的应用进程之间建立「端-端」连接，是弥补通信子网服务的不足，提高传输服务的可靠性和保证服务质量

网络层: 路由选择，寻址，拥塞控制，分段和重组, 解决的是由点-点链路组成的传输路径的路由选择与分组交付问题

数据链路层：差错控制

局域网中采用的两种访问控制策略: 争用型介质访问技术(CSMA/CD, 随机访问), 确定型介质访问(Token方式, 循环访问)

http2

HTTP 2.0把解决性能问题的方案内置在了传输层，通过多路复用来减少延迟，通过压缩 HTTP首部降低开销，同时增加请求优先级和服务器端推送的功能。 在 HTTP/2 中，同域名下所有通信都在单个连接上完成，这个连接可以承载任意数量的双向数据流。每个数据流都以消息的形式发送，而消息又由一个或多个帧组成。多个帧之间可以乱序发送，因为根据帧首部的流标识可以重新组装。

连接 -> 流 -> 消息 -> 帧

基于TCP有序字节流实现，因此应用层的多路复用并不能做到无序地并发，在丢包场景下会出现队头阻塞问题

http3

  • 解决队头阻塞
  • 解决tcp握手与tls握手无法合并的问题，基于tls1.3, 在http3的QUIC层中实现了连接复用

路由协议

路由器可以分析各种不同类型网络传来的数据包的目的地址，把非TCP/IP网络的地址转换成TCP/IP地址，或者反之；再根据选定的路由算法把各数据包按最佳路线传送到指定位置。

路由分为静态(手工配置)和动态路由，动态路由是基于动态路由协议实现的，不需要手动书写，路由器之间会互相学习，动态路由在一定程度上减少了管理任务，但也有缺点，会占用网络带宽。

动态路由算法分为距离矢量路由算法(基于Bellman-Ford)和链路状态路由算法(基于Dijkstra)

RIP基于UDP协议，BGP基于TCP协议，OSPF基于IP

route add [目标ip] mask [网络号] [网关地址] metric [最大跳数] # 静态路由设置

ip route add [目的ip | 10.176.48.0/20] via [下一跳的ip | 10.173.32.1] dev [出口网卡 | eth0]

ARP

用在局域网环境中，通过arp协议获取ip对应的mac地址

arp -s [ip地址] [mac地址] # 手工绑定ip与mac地址

arp -a # 显示所有接口的arp缓存表

arp -a -N [ip] # 显示指定ip的arp缓存表

RARP

用在局域网环境中，通过mac地址获取对应的局域网ip

RARP是一种逝去的地址分配技术，是Bootp和DHCP的鼻祖，目前我们的电脑基本不会用到这个协议，只有部分无盘工作站等情况需要用到。

IARP

IARP用于实现帧中继网络(用于广域网)中DLCI到IP地址的映射，生成帧中继映射表（类似ARP表），实现数据封装与通信

五类ip地址

  • A类地址(私有地址范围10.0.0.0到10.255.255.255是私有地址)的第一位固定为0，子网掩码255.0.0.0，最大主机数16777214
  • B类地址(私有地址范围172.16.0.0-172.31.255.255)的前两位固定为10，子网掩码255.255.0.0，最大主机数65534
  • C类地址(私有地址范围192.168.0.0-192.168.255.255)的前三位固定为110。 子网掩码分别为：255.255.255.0，最大主机数254
  • D类地址，前四位固定为1110，用于多播地址
  • E类地址，前四位固定为1111，保留使用

CIDR(无类型域间选路)

相较于五类型地址，子网掩码的设置更加自由

例如10.100.122.2/24, 这个地址表示前24位是网络号，后8位是主机号，所对应的子网掩码，255.255.255.0，广播地址为10.100.122.255。如果发送这个地址，所有10.100.122网络里面的机器都可以收到。

ICMP

请求报文和差错报文控制

raceroute 原理是利用 ICMP 的 ttl expired 通知机制，每次通过不断增加 ttl 从而不断发现下一跳路由，traceroute 发送的是端口号 > 30000 的数据报，所以到达目的主机的时候，会收到端口不可达的 ICMP 回应，这个时候源主机就知道主机可以连通了。

ping -l 200 [ip] # 指定TTL数据包的大小

ping [ip] -i 2 # 指定生存周期TTL为2，可以间接查看沿途经过的第2个路由器

pathping [host] # 跟踪数据包的路径，类似的命令还有tracert、traceroute

traceroute www.baidu.com

PPP

为在同等单元之间传输数据包这样的简单链路设计的链路层协议。这种链路提供全双工操作，并按照顺序传递数据包。设计目的主要是用来通过拨号或专线方式建立点对点连接发送数据，使其成为各种主机、网桥和路由器之间简单连接的一种共通的解决方案。当用户拨号接入 ISP 时，路由器的调制解调器对拨号做出确认，并建立一条物理连接（底层up）。PC 机向路由器发送一系列的 LCP 分组（封装成多个 PPP 帧）。 物理层

波特率，可以通俗的理解为一个设备在一秒钟内发送（或接收）了多少码元的数据。发送方每秒发送9600码元，接收方每秒仅能接受4800码元，并非无法接收，只能收到部分数据
