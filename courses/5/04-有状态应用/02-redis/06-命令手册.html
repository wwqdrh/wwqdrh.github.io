大部分命令成功时返回1，失败时返回0

Key

  • DEL: 删除单个字符串类型、单个列表、集合、有序集合或哈希表类型的 key
  • DUMP: 将key进行序列化
  • RESTORE: 将数据进行反序列化为key
  • EXISTS: 检查给定 key 是否存在
  • EXPIRE: 使用新的生存时间(秒)取代对应key旧的生存时间
  • PEXPIRE: 使用新的生存时间(毫秒)取代对应key旧的生存时间
  • EXPIREAT: 指定时间戳(秒级 10位)过期
  • PEXPIREAT: 指定时间戳(毫秒级 13位)过期
  • TTL: 返回剩余生命周期(秒), 不存在返回-2，存在但没有返回-1
  • PTTL: 返回剩余生命周期(毫秒)
  • KEYS: 查找所有符合模式的key
  • MIGRATE: 转移key到新的实例上
  • MOVE: 将当前数据库的 key 移动到给定的数据库 db 当中(可以当做锁的原语，已经存在则移动失败返回0)
  • OBJECT: 审查对象
    • REFCOUNT: 给定 key 引用所储存的值的次数。此命令主要用于除错
    • ENCODING: 返回给定 key 锁储存的值所使用的内部表示
    • IDLETIME: 返回给定 key 自储存以来的空转时间(idle， 没有被读取也没有被写入)，以秒为单位
  • PERSIST: 移除生存时间，转为持久性的
  • RANDOMKEY: 随机返回(不删除)一个 key
  • RENAME: 更名，两个名字相同返回错误，newkey已经存在时直接覆盖
  • RENAMENX: 仅当newkey不存在时才修改成功，key不存在时返回错误
  • SORT: 排序返回
  • TYPE: 返回 key 所储存的值的类型
  • SCAN
    • SCAN 命令用于迭代当前数据库中的数据库键。
    • SSCAN 命令用于迭代集合键中的元素。
    • HSCAN 命令用于迭代哈希键中的键值对。
    • ZSCAN 命令用于迭代有序集合中的元素（包括元素成员和元素分值）。

String

  • APPEND: 如果存在且是字符串加入成功返回字符串长度
  • BITCOUNT: 计算给定字符串中，被设置为 1 的比特位的数量
  • BITOP: 对一个或多个保存二进制位的字符串 key 进行位元操作，并将结果保存到 destkey 上
  • DECR: 将 key 中储存的数字值减一,如果值包含错误的类型，或字符串类型的值不能表示为数字，那么返回一个错误
  • DECRBY: 与DECR类似，不过是减n
  • INCR: 将 key 中储存的数字 (不存在时会初始化为0)值加一, 如果值包含错误的类型，或字符串类型的值不能表示为数字，那么返回一个错误
  • INCRBY: 与INCR类似，不过是加n
  • GET: 返回key所关联的值
  • GETBIT: 对 key 所储存的字符串值，获取指定偏移量上的位(bit)
  • GETRANGE: 返回[start, end]子字符串
  • GETSET: 将给定 key 的值设为 value ，并返回 key 的旧值(old value)
  • INCRBYFLOAT: 无论加法计算所得的浮点数的实际精度有多长， INCRBYFLOAT 的计算结果也最多只能表示小数点的后十七位
  • MGET: 返回所有(一个或多个)给定 key 的值, 不存在的key，值用nil代替
  • MSET: 同时设置一个或多个 key-value 对, 对于已经存在的会进行覆盖
  • MSETNX: 同时设置一个或多个 key-value 对，如果有一个已经存在那么所有的都会拒绝执行
  • SETEX: 将值 value 关联到 key ，并将 key 的生存时间设为 seconds。(SET key value + EXPIRE key seconds)
  • PSETEX: 与SETEX类似，不过是以毫秒为单位
  • SET: 将字符串值关联到key，会覆盖旧值
  • SETNX: 将 key 的值设为 value ，当且仅当 key 不存在
  • SETBIT: 对 key 所储存的字符串值，设置或清除指定偏移量上的位(bit)
  • SETRANGE: 用 value 参数覆写(overwrite)给定 key 所储存的字符串值, 从偏移量 offset 开始, 超出长度的对象使用zerobytes(\x00)进行填充
  • STRLEN: 返回 key 所储存的字符串值的长度

Hash

  • HDEL: 删除哈希表 key 中的一个或多个指定域，不存在的域将被忽略。
  • HEXISTS: 查看哈希表 key 中，给定域 field 是否存在。
  • HGET: 返回哈希表 key 中给定域 field 的值。
  • HGETALL: 返回哈希表 key 中，所有的域和值。
  • HINCRBY: 为哈希表 key 中的域 field 的值(仅限数字表达式)加上增量 increment(可以是负数)
  • HINCRBYFLOAT: 为哈希表 key 中的域 field 加上浮点数增量 increment
  • HKEYS: 返回哈希表 key 中的所有域。
  • HVALS: 返回哈希表 key 中所有域的值
  • HLEN: 返回哈希表 key 中域的数量。
  • HMGET: 返回哈希表 key 中，一个或多个给定域的值。
  • HMSET: 同时将多个 field-value (域-值)对设置到哈希表 key 中。(会进行覆盖)
  • HSET: 将哈希表 key 中的域 field 的值设为 value 。(会进行覆盖)
  • HSETNX: 将哈希表 key 中的域 field 的值设置为 value。(如果存在则无效，返回0)
  • HSCAN: 命令用于迭代哈希键中的键值对

List

  • BLPOP: LPOP的阻塞版本，弹出左边元素，直到超时或发现可弹出元素为止
  • BRPOP: RPOP的阻塞版本，弹出右边元素，直到超时或发现可弹出元素为止
  • RPOPLPUSH: 将列表 source 中的最后一个元素(尾元素)弹出返回给客户端并加到新的列表的表头
  • BRPOPLPUSH: RPOPLPUSH的阻塞版本，假如在指定时间内没有任何元素被弹出，则返回一个 nil 和等待时长。否则返回弹出的元素以及等待时长
  • LINDEX: 返回列表 key 中，下标为 index 的元素
  • LINSERT: (可以指定BEFORE、AFTER)将值 value 插入到列表 key 当中，位于值 pivot 之前或之后
  • LLEN: 返回列表的长度
  • LPOP: 移除并返回列表 key 的头元素
  • RPOP: 移除并返回列表 key 的尾元素
  • LPUSH(返回执行后的列表长度): 将一个或多个值 value 插入到列表 key 的表头
  • LPUSHX: 将值 value 插入到列表 key 的表头，当且仅当 key 存在并且是一个列表, (当key不存在时什么也不做)
  • RPUSH: 将一个或多个值 value 插入到列表 key 的表尾
  • RPUSHX: 将值 value 插入到列表 key 的表尾，当且仅当 key 存在并且是一个列表
  • LRANGE: 返回列表 key 中指定区间内的元素，区间以偏移量 start 和 stop 指定。可以使用负数下标
  • LREM: 根据参数 count(大于0为正向 小于0为反向)的值，移除列表中与参数 value 相等的元素
  • LSET: 将列表 key 下标为 index 的元素的值设置为 value
  • LTRIM: 只保留[start, stop]之间的元素

Set

  • SADD: 将一个或多个 member 元素加入到集合 key 当中，已经存在于集合的 member 元素将被忽略
  • SCARD: 返回集合 key 的基数(集合中元素的数量)
  • SDIFF: 返回一个集合的全部成员，该集合是所有给定集合(不存在的集合为空集)之间的差集
  • SDIFFSTORE: 类似SDIFF, 不过会再将结果保存在新的集合中
  • SINTER: 返回一个集合的全部成员，该集合是所有给定集合的交集
  • SINTERSTORE: 类似SINTER, 不过会再将结果保存在新的集合中
  • SUNION: 返回一个集合的全部成员，该集合是所有给定集合的并集
  • SUNIONSTORE: 类似SUNION, 不过会再将结果保存在新的集合中
  • SISMEMBER: 判断 member 元素是否集合 key 的成员
  • SMEMBERS: 返回集合 key 中的所有成员
  • SMOVE: 将 member 元素从 source 集合移动到 destination 集合
  • SPOP: 移除并返回集合中的一个随机元素
  • SRANDMEMBER: 返回集合中的一个随机元素(不删除)
  • SREM: 移除集合 key 中的一个或多个 member 元素，不存在的 member 元素会被忽略
  • SSCAN: 迭代集合键中的元素

SortedSet

  • ZADD: 将一个或多个 member 元素及其 score 值加入到有序集 key 当中，如果某个 member 已经是有序集的成员，那么更新这个 member 的 score 值，并通过重新插入这个 member 元素，来保证该 member 在正确的位置上。
  • ZCARD: 返回有序集 key 的基数
  • ZCOUNT: 返回有序集 key 中， score 值在 min 和 max 之间(默认包括 score 值等于 min 或 max )的成员的数量。
  • ZINCRBY: 为有序集 key 的成员 member 的 score 值加上增量 increment 。
  • ZRANGE: 返回[start, stop]之间的值，按照score升序以及字典序进行排序
  • ZREVRANGE: 返回[start, stop]之间的值，按照score降序以及字典序进行排序
  • ZRANGEBYSCORE: min 和 max 可以是 -inf 和 +inf ，这样一来可以在不知道最低和最高 score 值的情况下使用
  • ZRANK: 返回有序集 key 中成员 member 的排名。其中有序集成员按 score 值递增(从小到大)顺序排列
  • ZREM: 移除有序集 key 中的一个或多个成员，不存在的成员将被忽略
  • ZREMRANGEBYRANK: 移除[start, end]区间内排名的元素
  • ZREMRANGEBYSCORE: 移除[start, end]区间内score的元素
  • ZREVRANGEBYSCORE: 返回[min, max]所有成员，按照score降序以及字典序排列
  • ZREVRANK: 返回有序集 key 中成员 member 的排名。其中有序集成员按 score 值递减(从大到小)排序
  • ZSCORE: 返回有序集 key 中，成员 member 的 score 值
  • ZUNIONSTORE: ZUNIONSTORE destination numkeys key [key ...] [WEIGHTS weight [weight ...]] [AGGREGATE SUM|MIN|MAX], 将numkeys个数的key聚合起来存储到destination中，aggregate提供聚合的方式(相加、最大、最小)，weights是乘法因子，在参与聚合之间会先处理乘法因子
  • ZINTERSTORE: 与ZUNIONSTORE类似，不过这里是存交集
  • ZSCAN: 命令用于迭代有序集合中的元素（包括元素成员和元素分值）。

Stream

  • xadd: 追加消息, 返回生成的消息ID， * 号表示服务器自动生成 ID，后面顺序跟着一堆 key/value, xadd codehole * name laoqian age 30
  • xdel 删除消息，这里的删除仅仅是设置了标志位，不影响消息总长度
  • xrange 获取消息列表，会自动过滤已经删除的消息
  • xread 消息长度Redis 设计了一个单独的消费指令 xread，可以将 Stream 当成普 通的消息队列 (list) 来使用。使用 xread 时，我们可以完全忽略消费组 (Consumer Group) 的存在，就好比 Stream 就是一个普通的列表 (list)。
  • xlen
  • del 删除 Stream
