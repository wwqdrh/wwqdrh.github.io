## LVM

LVM是Linux环境中对磁盘分区进行管理的一种机制，是建立在硬盘和分区之上、文件系统之下的一个逻辑层，可提高磁盘分区管理的灵活性。

- PE: 物理扩展
- PV: 物理卷
- VG: 卷组
- LV: 逻辑卷

> 将物理硬盘格式化成PV
> 创建一个VG
> 基于VG创建最后要使用的LV
> 将创建好的LV进行文件系统的格式化，然后挂载使用

Linux 大容量磁盘分区软件

`sudo apt-get install lvm2`


创建物理卷: pvcreate

查看物理卷: pvs
```bash
streama@ubuntu:~$ sudo pvs
  PV         VG     Fmt  Attr PSize PFree
  /dev/sdb1  vgdata lvm2 a--  3.60t    0
```

创建卷组: vgcreate

```bash
sudo vgcreate -s 32M [name] /dev/sda3 // 基于/dev/sda3这个物理卷创建PE大小为32M的卷组（注意这里是把pv大小全部变成pe然后加到卷组中）
```

查看卷组: vgs
```bash
streama@ubuntu:~$ sudo vgs
  VG     #PV #LV #SN Attr   VSize VFree
  vgdata   1   1   0 wz--n- 3.60t    0
```

创建逻辑卷

```bash
sudo lvcreate -L 15G -n [逻辑卷名字] [卷组名字]
```

查看逻辑卷: lvdisplay

```bash
streama@ubuntu:~$ sudo lvdisplay
```

格式化逻辑卷

> 当我们创建 LV nickvg/nicklv00 时，其实是创建了名称为 /dev/nickvg/nicklv00 的设备文件, 需要格式化后才能使用，常用的是ext4

```bash
sudo mkfs.ext4 /dev/nickvg/nicklv00
```

激活逻辑卷: vgchange -ay <逻辑卷路径>

```bash
streama@ubuntu:~$ sudo vgchange -ay /dev/vgdata/lvdata
LV Status available
```

挂载逻辑卷：mount <逻辑卷名> <挂载目录>

```shell
mount /dev/vgdata/lvdata /data
```

设置开机自动挂载

```bash
# /etc/fstab

/dev/mapper/nickvg-nicklv00 /data       ext4    defaults       0       0
```
