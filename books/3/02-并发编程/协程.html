超时协程退出问题

无法强制kill一个协程

  • 1、尽量使用非阻塞I/O
  • 2、业务逻辑总是考虑退出机制
  • 3、任务分段执行，超时后即使退出

限制协程并发数量

使用一个固定长度的通道来实现，当一个协程执行完成之后会从中读取出来一个，然后下一个协程就可以使用通道并执行了

func LimitGoroutineNumber() {
	var wg sync.WaitGroup // 用于等待最后三个
	ch := make(chan struct{}, 3)
	for i := 0; i < 10; i++ {
		ch <- struct{}{} // 当写了三次就不能再写了，除非下文有程序执行完了能够继续写入
		wg.Add(1)
		go func(i int) {
			defer wg.Done()
			log.Println(i)
			time.Sleep(time.Second)
			<-ch
		}(i)
	}
	wg.Wait()
}
