适用于有重叠子问题和最优子结构性质的问题，一旦一个子问题的求解得到结果，以后的的计算过程就不会修改它，即无后效性

求解问题的过程形成了一张有向无环图。动态规划只解决每个子问题一次，具有天然剪枝的功能，从而减少计算量

分为自底向上(递推，迭代)和自顶向下(记忆化，递归)的两种解决问题的方式

# 背包问题

## 01背包

已知第i个物品的重量w(i), 价值v(i), 背包的总容量W

设f(i, j)为前i个物品的情况下，容量为j的背包所能达到的最大总价值

```go
for i := 1; i <= n; i++ {
	l := W
	for l >= w[i] {
		// 当前的容量满足
		f[l] = max(f[l], f[l - w[i]] + v[i])
		l -= w[i]
	}
}
```

## 完全背包

与01背包类似，不过这里的物品能够取无限次

> 状态转移方程容易与01背包弄混

```go
for i := 1; i <= n; i++ {
	for l := w[i]; l <= W; l++{
		f[l] = max(f[l], f[l - w[i]] + v[i])
	}
}
```

## 多重背包

与01背包类似，不过这里物品i能够选择的次数为k(i)

可以将`每种物品选k(i)次`等价转换为`有k(i)个相同的物品，每个物品选一次`

## 混合背包

前三种混合，有点只能取一次，有的取无限次，有的取k次（其实就像多重背包一样，每种物品i能够取指定的k(i)次）

## 二维费用背包

相较于前面的背包的价值维度，然后新增一个维度，例如打包的时间，在状态中增加一维存放

## 分组背包

在所有物品中选择一件变成了从当前组中选择一件。

## 有依赖的背包

对于一个主件和它的若干附件，选择了主件就必须选择附件，可以看做分组背包

## 泛化物品的背包

没有固定的费用和价值，它的价值是随着分配给它的费用而定

将固定的价值转换成函数的引用即可。

## 石子游戏ix

给一个数组表示石子的价值，当谁拿出来导致总拿出来的变为3的倍数，那么就输了，如果拿空了，Bob直接获胜

-   Alice先手
-   如果拿完没有分出胜负，Bob获胜

由于和只与3有关，取模分为 0 1 2三种石子，0的可以用来换手，1121212… 2212121… 两种情况。

```go
// 返回alice能否获胜
func stoneGameIX(stones []int) bool {
    cnt1, cnt2, cnt3 := 0, 0, 0
    for _, item := range stones {
        t := item % 3
        switch t {
        case 0:
            cnt1++
        case 1:
            cnt2++
        case 2:
            cnt3++
        }
    }
    if cnt1%2 == 0 {
        return cnt2 >= 1 && cnt3 >= 1 // 1少些先拿1 2少些先拿2能够保证必赢
    }

    return cnt2-cnt3 > 2 || cnt3-cnt2 > 2  // 存在换一次手的可能，必须大于2才能保证不是因为拿完了导致的bob胜利，这样换了一次手能够保证alice胜利
}
```

## 删除连续的字符串

由`A B`组成的字符串，如果存在三个及以上连续的A，A就可以删除中间的部分，B同理，不能删除对方的，双方都采用最优策略看谁先赢

#贪心

由于AB的删除操作不会互相影响那么就只需要统计下各自能操作的次数就行

```go
func winnerOfGame(colors string) bool {
	freq := [2]int{}
	cur, cnt := 'C', 0
	for _, c := range colors {
		if c != cur {
			cur, cnt = c, 1
		} else {
			cnt++
			if cnt >= 3 {
				freq[cur-'A']++
			}
		}
	}
	return freq[0] > freq[1]
}
```
